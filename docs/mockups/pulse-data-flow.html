Where this lives

Three realistic placements for the same animation system — one build, tuned per surface.

Placement 01Landing-page hero

Runs full-bleed behind the headline on the public Pulse page, pointer-events off, at ~60% of this intensity. The scenario auto-rotates every 20s so the page never looks the same twice.

Placement 02In-app loading & empty states

A single-stream slice of the same system (one source → core → one surface) replaces spinners while dashboards hydrate — the wait itself tells the product story.

Placement 03Sales deck & wizard finale

Full interactive version with the scenario tabs, used live in pitches and as the closing screen of the client setup wizard — "this is what we just switched on for you."

Implementation notes: one <canvas> for the particle streams + plain DOM cards for the toasts — no libraries, ~10 KB gzipped. Respects prefers-reduced-motion (static paths, slower toast cycle), throttles to zero when the tab is hidden, and the whole stage collapses to a vertical top→down flow on phones. All figures shown are example data.
